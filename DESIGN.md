# 日本語リーダー（仮称：Yomu）— 設計・企画ドキュメント

Satori Reader ライクな日本語学習リーダーの Web アプリ。難しい単語はホバー/クリックで
意味・例文を表示し、文末ボタンで英訳を表示、振り仮名はワンタップで表示/非表示を切り替える。

- **形態**: Web アプリ（レスポンシブ、PC/タブレット/スマホ対応）
- **コンテンツ**: 形態素解析で自動的に下地を作り、後から補正する方式
- **ステータス**: 設計フェーズ（2026-07-09 時点）

---

## 1. 目的・ターゲット

- **ターゲット**: 中級までの日本語学習者（漢字と語彙で詰まる層）。
- **価値**: 「辞書を引く手間」をゼロにし、実際の文章を読み続けられる状態を作る。
- **差別化**: Satori Reader は手動注釈で質が高いが有料・コンテンツ固定。こちらは
  **自分の好きな文章を貼ると自動で読める形になる**方向を狙える。

---

## 2. コア機能（MVP）

| 機能 | 内容 | 優先度 |
|---|---|---|
| 振り仮名トグル | `<ruby>` の読み仮名を全体で表示/非表示 | 必須 |
| 単語ポップアップ | 単語をホバー/クリックで 読み・品詞・意味・例文 を表示 | 必須 |
| 文単位の英訳 | 各文末ボタンで英訳を表示 | 必須 |
| 文章の取り込み | テキストを貼る→自動で単語分割・読み・意味付与 | 必須 |
| 音声読み上げ | 文・単語の TTS 再生 | 後回し（Phase 2） |
| 単語帳 | 「単語帳に追加」→ 復習リスト・SRS | 後回し（Phase 2） |
| ユーザー登録/同期 | 進捗・単語帳をアカウントに保存 | 後回し（Phase 3） |

## 3. コンテンツ生成パイプライン（本丸）

UI より「文章をどう単語に割り、意味を紐づけるか」が品質を決める。

```
生テキスト
   │  ① 形態素解析（kuromoji.js / MeCab）
   ▼
トークン列（表層形・読み・品詞・基本形）
   │  ② 辞書引き（JMdict：無料の日英辞書）
   ▼
語ごとの候補（意味・品詞・例文）
   │  ③ 補正（任意：AI or 手動で文脈に合う語義を選ぶ）
   ▼
注釈済みドキュメント（JSON）→ フロントで描画
```

- **① 形態素解析**: [kuromoji.js](https://github.com/takuyaa/kuromoji.js) はブラウザ/Node で動く。
  読みガナ（振り仮名の元）と基本形が取れる。サーバ側なら MeCab + UniDic がより高精度。
- **② 辞書**: [JMdict](https://www.edrdg.org/jmdict/j_jmdict.html)（日英）。ライセンス表記に注意。
- **③ 補正**: 慣用句（例「声をかける」）や文脈依存の語義は自動だと外すため、
  Claude API で「この文でのこの語の意味」を選ばせる、または手動で直せる管理画面を用意。
- **英訳**: 文単位の対訳は自動解析では作れない。Claude API で文ごとに生成するのが現実的。

### 難所メモ
- 複合表現・慣用句の区切り（「〜たり〜たり」「声をかける」）。
- 同じ表層でも文脈で読み/意味が変わる語。
- 送り仮名を含む活用（手伝っ**た** → 基本形「手伝う」に正規化して辞書を引く）。

## 4. データモデル（案）

```jsonc
// Document
{
  "id": "kona-02",
  "title": "子どもたちと過ごす日々",
  "subtitle": "幼稚園の先生の一日",
  "sentences": [
    {
      "id": "s1",
      "translation": "I usually get to work a little before 8.",
      "tokens": [
        { "surface": "出勤", "reading": "しゅっきん", "base": "出勤",
          "pos": "名詞", "entryId": "jmdict:1338260", "annotated": true },
        { "surface": "は", "reading": "は", "pos": "助詞", "annotated": false }
        // ...
      ]
    }
  ]
}

// DictionaryEntry
{
  "id": "jmdict:1338260",
  "word": "出勤",
  "reading": "しゅっきん",
  "pos": "名詞",
  "glosses": ["going to work", "attendance"],
  "examples": [{ "ja": "毎朝8時に出勤する。", "en": "I go to work at 8 every morning." }]
}
```

- `annotated:false` の語（助詞・句読点など）はクリック不可・振り仮名なしにできる。
- 振り仮名は各トークンの `reading` から `<ruby>` を生成。

## 5. 技術スタック（案）

| 層 | 選択肢 | 備考 |
|---|---|---|
| フロント | React + Vite / Next.js | Satori 風 UI。ルビ・ポップアップは上の試作の延長 |
| 形態素解析 | kuromoji.js（軽量）/ サーバ MeCab（高精度） | まずは kuromoji で十分 |
| 辞書 | JMdict を前処理して SQLite/JSON | 語→意味の索引を事前生成 |
| AI 補正・英訳 | Claude API（claude-opus-4-8 / sonnet） | 文脈語義選択＆文対訳生成 |
| 保存 | Phase1 はローカル(IndexedDB)、後で Supabase 等 | 段階導入 |

---

## 6. UI 方針

- 上の試作の路線（青い下線＝クリック可能語、文末に翻訳/再生ボタン、右上に振り仮名トグル）。
- モダンでミニマル。読書中は注釈を目立たせすぎない（下線は控えめ、ポップアップは軽い影）。
- 振り仮名は「全体トグル」に加え、将来的に「未知語だけ表示」モードもあり得る。
- モバイルではホバーがないのでタップ主体。ポップアップは画面内に収める。

---

## 7. フェーズ別ロードマップ

**Phase 0 — 動く試作（Web）**
- テキストを貼る → kuromoji で分割 → ルビ付き表示 → 振り仮名トグル。
- 単語クリックで JMdict の意味をポップアップ。
- ゴール: 「自分の文章が読める形になる」体験の確認。

**Phase 1 — 実用リーダー**
- 文単位の英訳（Claude API）。
- 慣用句・語義の補正フロー。
- 記事の保存・一覧（ローカル）。

**Phase 2 — 学習機能**
- 音声読み上げ（TTS）。
- 単語帳＋SRS 復習。

**Phase 3 — サービス化**
- アカウント・進捗同期。
- コンテンツ配信 or ユーザー投稿。
- 課金（フリーミアム/サブスク：他アプリと同じ路線）。
